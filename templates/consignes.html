Atelier — Automatisation des tests d'API publiques

Vous allez choisir une API dans la liste public-apis et construire une mini-solution de tests automatisés déployée sur PythonAnywhere.

📊 Voir le Dashboard →

1) Objectifs & livrable final

ObjectifTester automatiquement une API (contrat + robustesse + QoS) et publier un dashboard.
LivrableUn dépôt GitHub + une app Flask sur PythonAnywhere + un historique de runs en SQLite.
ContraintesTests non destructifs, charge limitée, clés API (si besoin) via Secret Github.

2) Plan de tests attendu

A. Tests "Contrat" (fonctionnels)

  • HTTP 200/204 attendu
  • Content-Type JSON
  • Champs obligatoires présents
  • Types (string/int/float/bool/list/object)
  • Cas d'entrée invalide → code 400/404/… attendu

B. Robustesse & QoS

  • Timeout (3s) + 1 retry max
  • Gestion 429 : backoff / attente
  • Latence : avg + p95
  • Taux d'erreur + disponibilité
